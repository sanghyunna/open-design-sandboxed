Stop asking

Ask instead: who owns the failure?

The cost of an AI mistake is no longer one bad response. It can create 300 tickets, open 80 PRs, or send 5,000 emails at once.

That is why testing, acceptance, safety, and risk control will be the most valuable capabilities over the next three years.

risk levels · 3 levels03 / 08

Three levels, three responses

L1 · green

Reversible

Write drafts, generate images, draft documents.
A mistake is Ctrl+Z, at zero cost.
Policy: let it run

L2 · amber

Partly reversible

Send draft emails, open PRs, edit staging data.
A mistake needs an apology or rollback.
Policy: human review

L3 · red

Irreversible

Send real emails, pay money, delete databases or production data.
A mistake is final.
Policy: hard block + two-person review

Never let an agent upgrade itself

An L1 task must not turn itself into L2. Authorization must be explicit, revocable, and time-bound.

policy as code04 / 08
Do not govern rules with documents · govern them with code

Thirty lines of YAML,
hard stops for red lines

# safety-policy.yaml · compiled → runtime guard
level_1_allow:
  - tools: [write_draft, generate_image, read_docs]

level_2_require_review:
  - tools: [send_email_draft, open_pr, write_staging_db]
    reviewer: human

level_3_hard_block:
  - tools: [send_real_email, transfer_money, delete_prod]
    unless: two_human_sign_off AND within_24h

forbidden_always:
  - "rm -rf /"
  - "drop table"
  - "force push origin main"

red-team checklist06 / 08

Seven required questions before launch

✓
Can it delete anything? Is there human review? Can it roll back in 60 seconds?
✓
Can prompt injection make it exceed its authority? (Run red-team prompts.)
!
Does it handle PII? Does its log contain PII too?
✓
When an upstream or downstream dependency fails, will it modify unrelated resources?
!
Will 100 agents running concurrently deadlock?
✓
Can it stop immediately when wrong? (Does the kill switch take effect within two seconds?)
!
Is someone on call during an incident? Does the runbook have an agent-specific section?
