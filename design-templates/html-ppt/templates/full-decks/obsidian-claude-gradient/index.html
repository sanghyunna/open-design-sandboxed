03 / 08
● COMPARE

Notion vs Obsidian · AI readiness

NOTION

Database-native

Good for structured tasks and teams, but:
• AI must use an API and cannot read live full text
• Nested blocks are complex and token-expensive
• Weak local ownership prevents durable memory

OBSIDIAN

Plain Markdown + links

Naturally AI-friendly:
• Everything is a file Claude can read directly
• Links form a graph, so extraction is nearly free
• Offline, Git-friendly, diffable, and reversible

💡 Key insight: AI does not need a smarter database; it needs a file system it can read itself.
04 / 08
● SETUP · 4 STEPS

From zero to the first AI-written note

1

Install Obsidian and enable the Local REST API plugin

A community plugin enabled with one check box. It lets external processes read and write your vault.

2

Connect Claude Desktop and the obsidian-mcp server

One MCP config file connects it; use the vault API key as the token.

3

Install five Obsidian skills

markdown / bases / canvas / cli / defuddle — teach Claude to use Obsidian correctly.

4

Let Claude organize once

“Merge duplicate concepts from my ten latest notes and create a new MOC” — results in 90 seconds.

05 / 08
● MCP CONFIG

claude_desktop_config.json

// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-obsidian"],
      "env": {
        "OBSIDIAN_API_KEY": "xxxxxxxxxxxxxxxx",
        "OBSIDIAN_HOST": "http://127.0.0.1:27123"
      }
    }
  }
}

Restart Claude Desktop and enter /mcp; Obsidian will show as connected.
