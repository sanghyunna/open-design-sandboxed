// part 02 · presenter view

Presenter view needs four pieces of information

①
Large current-slide view

Takes up more than half the view, so a glance tells you exactly what the audience is seeing.

②
Next-slide preview

Lets you prep the transition line in advance, so you never forget what's next.

③
Script area

Large font, high contrast, scrollable — this is what the presenter is actually looking at.

④
Timer + page number

Knowing how long you've talked and how many pages remain is what pacing depends on.

Presenter mode should give you four pieces of information, ranked by importance.

First, a large current-slide view. It has to take up more than half the screen, because it's your 'sync anchor' with the audience — whatever they see, you need to see too.

Second, a next-slide preview. A lot of people don't get why this matters: the moment a talk stalls isn't mid-slide, it's the 2 seconds after you flip to the next one. If you can already see what's coming, the transition line writes itself.

Third, the script area — today's focus, and I'll dig into it next. One hard requirement up front: large font, high contrast, scrollable. You're only glancing at it peripherally while talking, so small text is unreadable in time.

Fourth, timer and page count. Knowing how long you've talked and how many slides remain is what pacing depends on. Keynote does this best; reveal.js's default isn't clear enough.

All four are essential. I've built all four into today's deck — press S and try it yourself.

// part 03 · script

3 iron rules for writing a script

01
Not a word-for-word manuscript — a "prompt signal"

Bold the key points, put transitions in their own paragraph, spell out data and names clearly — so a glance is all you need to pick up.

02
150–300 characters per slide, no more, no less

Under 150 and the prompt isn't enough; over 300 and you won't have time to read it. Pace it at 2–3 minutes per slide.

03
Write in spoken language, not formal writing

"Therefore" → "so"; "said plan" → "this plan." Read it aloud as you write — it should sound like speech.

Now the core part — how to write a script. I've boiled it down to 3 iron rules.

Rule one: a script isn't a manuscript. When people hear 'script' they assume writing out every sentence verbatim. Wrong. If you read it word for word, the audience will notice instantly and your credibility tanks.

A script's real job is to be a 'prompt signal' — bold the key points, put transitions in their own line, spell out data and proper nouns. That way a glance is enough to pick up, but what comes out is still your own words.

Rule two: keep each slide to 150–300 characters. This is a number I arrived at after a dozen-plus talks. Fewer than 150 and the prompt isn't enough — you'll stall mid-sentence; more than 300 and you can't scan it in time. At 2–3 minutes per slide, this range is right.

Rule three: write in spoken language. This is where most people trip up. If you write 'therefore' formally, you'll actually say 'so'; if you write 'said plan,' you'll say 'this plan.' Read it aloud as you write — it should sound natural, not stilted.

Put these three together and giving a talk suddenly feels effortless.
