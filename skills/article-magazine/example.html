After Reading @trq212,
I Replaced All My Markdown with HTML

In the age of AI writers, editors, and agents, Markdown is no longer the final reader-facing format. HTML is.

Tom Pan·May 11·4 min read

Three observations that convinced me

First, Markdown is pleasant to write, but readers never voted for it; they receive a platform-owned renderer. Second, Markdown loses when shared as a screenshot, while HTML can be a finished visual. Third, every publisher interprets Markdown differently; inline HTML and CSS retain the intended result.

HTML is verbose, and that is true

Writing <div class="..."> repeatedly is tedious. The same content once took 30 seconds in Markdown and 30 minutes in HTML.

AI reduces those 30 minutes to 30 seconds. You own the final form; AI handles the repetitive details.

We also built a tool

Inspired by the post and the Claude Code team's practice, we built HTML Anything. Paste Markdown, CSV, or JSON, choose a magazine, deck, poster, carousel, or data-report template, and press ⌘+Enter. A signed-in Claude, Cursor, or Codex session generates share-ready HTML; revisions run only the diff.

Conclusion

If manual editor reformatting feels wasteful, read the original post and try a tool that promotes Markdown to HTML automatically.
