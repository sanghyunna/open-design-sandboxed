## Was ist Open Design

рҹҺЁ **Die local-first, quelloffene Alternative zu [Claude Design][cd].** &nbsp;рҹ–ҘпёҸ **Native Desktop-App fГјr macOS und Windows.** &nbsp;вҡЎ **100+ Skills** В· вңЁ **150 markenreife `DESIGN.md`-Systeme** В· рҹ“Ұ **261 sofort einsetzbare Plugins.** &nbsp;рҹ–јпёҸ Erzeugt **Web- В· Desktop- В· Mobile-Prototypen**, **Live-Dashboards / Artefakte**, **PrГӨsentationen**, **Bilder**, **Videos** sowie **HyperFrames**-Motion-Graphics. рҹ”’ Sandboxed iframe-Vorschau В· Export als HTML / PDF / PPTX / MP4. &nbsp;рҹӨ– **LГӨuft auf Claude Code В· OpenClaw В· Codex В· Cursor В· OpenCode В· Qwen В· Copilot В· Hermes В· Kimi В· Antigravity und 21 lokalen CLIs**, oder Гјber BYOK an jedem OpenAI-kompatiblen Endpunkt.

Open Design ist das, was entsteht, wenn die **agent-native** Schleife, die Anthropic mit Claude Design eingefГјhrt hat вҖ” das Briefing erkunden, die Richtung festlegen, das Artefakt streamen, kritisieren, ausliefern вҖ” nicht lГӨnger geschlossen ist und zu einem **Dateisystem aus Skills, Designsystemen und Plugins** wird, das die Coding-Agents, die schon auf deinem Laptop liegen, lesen, schreiben und neu kombinieren kГ¶nnen. Dein CLI wird zur Design-Engine, dein Laptop zum Studio und die `DESIGN.md` deines Teams zum Markenvertrag.

Es ist auГҹerdem die **Figma-Alternative fГјr das Agent-Zeitalter** вҖ” statt Pixel auf einer Leinwand zu schieben, liefert es einseitige Artefakte in echtem CSS, echten Schriften, echten Komponenten, direkt exportiert als HTML / PDF / PPTX / MP4 вҖ” bereits durch dein Designsystem geformt, bereits lauffГӨhig in dem Agent, den du jeden Tag verwendest.

## Warum Open Design

> **Im April 2026 verГ¶ffentlichte Anthropic [Claude Design][cd] вҖ” das erste Mal, dass ein LLM aufhГ¶rte, Prosa zu schreiben, und begann, Design-Artefakte direkt auszuliefern.** Es ging viral. Aber es blieb closed-source, nur kostenpflichtig, nur cloudbasiert, gebunden an Anthropics Modell, Anthropics Skills, Anthropics OberflГӨche. ## Open Design aus deinem Coding-Agent nutzen

Open Design liefert einen **stdio-MCP-Server** und **Installationsskripte pro Agent** aus. Jeder MCP-kompatible Agent in einem anderen Repo kann Dateien aus deinen lokalen Open-Design-Projekten direkt lesen вҖ” Token-CSS, JSX-Komponenten, Einstiegs-HTML вҖ” als eine strukturierte, nach Namen abfragbare API. ```bash

**Warum MCP?** Bei jeder Iteration ein ZIP zu exportieren und neu anzuhГӨngen, bricht den Flow. MCP legt die Designquelle direkt offen вҖ” der Agent sieht immer die Live-Datei.

**FГјr einen Agent, der bei null startet,** platziert der Installer `~/.config/<agent>/open-design.json` (oder das Plattform-Г„quivalent) plus ein Copy-Paste-MCP-Snippet. Cursor bekommt einen Ein-Klick-Deeplink; Claude Code bekommt einen `claude mcp add-json`-Einzeiler; jeder andere Agent bekommt JSON in dem Schema, das seine Konfiguration erwartet. VollstГӨndiger Ablauf pro Agent вҶ’ **Settings вҶ’ MCP server** in der Desktop-App, oder [`docs/agent-adapters.md`](../../docs/agent-adapters.md).

**Sicherheitsmodell.** StandardmГӨГҹig schreibgeschГјtzt, der Daemon bindet an `127.0.0.1`, und SSRF wird am Proxy-Rand blockiert.
