● COMPARE

Notion vs Obsidian · AI-Friendliness

NOTION

Database-native

Great for structured tasks and team collaboration, but —
• AI has to go through the API and can't get real-time full text
• nested block structures are complex and token-expensive
• poor local storage means it can't serve as long-term memory

OBSIDIAN

Pure Markdown + Bi-links

Naturally AI-friendly —
• everything is just a file Claude can read directly
• bi-links form a natural graph, so entity extraction is nearly free
• offline, git-able, diffable, and revertible

💡 Key insight: AI doesn't need a 'smarter database' — it needs a 'file system it can read itself.'

● SETUP · 4 STEPS

From zero to your first 'AI writes a note'

1

Install Obsidian + enable the Local REST API plugin

A community plugin — one checkbox turns it on. It lets external processes read/write your vault.

2

Connect Claude Desktop + the obsidian-mcp server

One config file wires up MCP — just fill in your vault's API key as the token.

3

Install the 5 obsidian-skills

markdown / bases / canvas / cli / defuddle — so Claude knows how to use Obsidian correctly.

4

Let Claude do a pass of its own

"Merge the duplicate concepts across my last 10 notes and generate a new MOC" — done in 90 seconds.

05 / 08
● MCP CONFIG

claude_desktop_config.json

// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-obsidian"],
      "env": {
        "OBSIDIAN_API_KEY": "xxxxxxxxxxxxxxxx",
        "OBSIDIAN_HOST": "http://127.0.0.1:27123"
      }
    }
  }
}

Restart Claude Desktop, type /mcp, and you'll see obsidian is connected.
