After reading @trq212's tweet,
I converted all my markdown to HTML

In short: in the age of AI writing, editors, and agents, markdown as an 'intermediate format' can't hold up anymore — HTML is the true reader-facing final form.

Tom Pan ·May 11 ·4 min read

Three observations that won me over

First, our love for markdown is mostly about how satisfying it is to write. But readers never got a vote. What readers get is always whatever a markdown renderer spits out — and that renderer belongs to the platform, not you.

Second, markdown loses when it comes to tweeting a screenshot. Screenshot any markdown snippet and you get a gray-and-white block flattened by GitHub's default theme. HTML can look like wallpaper art.

Third, WeChat / Zhihu / Xiaohongshu / Notion / Feishu each interpret markdown differently. Write it once and you'll adjust it five times for five platforms. HTML + inline CSS: paste once, renders identically everywhere.

But HTML is verbose — that part's true

<div class="..."> Writing enough of it makes you sick of it — that part's real. Nobody used to want to spend the effort on HTML, because for the same content markdown takes 30 seconds and HTML takes 30 minutes.

The variable is — AI brought those 30 minutes down to 30 seconds.
You write markdown, AI upgrades it into deliverable HTML. You own the final form, AI handles the tedious details.

So we built a tool while we were at it

Inspired by the original tweet, plus what the Claude Code team was already doing, we built HTML Anything. Takeaway

If you also feel like 'markdown → manually reformatting in an editor' is wasting your life — check out the original tweet, check out the Claude Code team's migration, then try any tool that automatically upgrades markdown into HTML.
